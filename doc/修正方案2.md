德州扑克中，EV 是与底池（Pot）大小严格绑定的。如果在单次加注底池（SRP，底池约 5.5bb）中，某个手牌的 DLEv 偏差是 3bb；到了 3-Bet 底池（底池约 20bb）中，它的偏差绝对不止 3bb。

更重要的是，有效后手与底池比（SPR - Stack-to-Pot Ratio）是决定深度限制误差的核心变量：

* **SPR 越小（如河牌前全下）**：未来博弈树越短，DLEv 的误差越趋近于 0。
* **SPR 越大（如 100bb 深筹 SRP）**：隐含赔率（Implied Odds）和反向隐含赔率（Reverse Implied Odds）的作用被无限放大，DLEv 的浅层计算误差最大。

因此，引入 **SPR 动态乘数（ Multiplier-based on SPR ）** 是工业级 Solver 修正的终极形态。我们把修正公式升级为：


$$EV_{Corrected} = DLEv \times EQR(SPR, \text{Texture}, \text{Hand Class})$$

基于你之前提供的数据（假设原数据背景为典型的中等 SPR 博弈，如 SPR $\approx 4 \sim 6$），我为你重新设计了 **8 种牌面的乘法修正函数**。

### 1. 三条面 (Trips)

* **核心逻辑：** 纯空气牌的乘数直接趋近于 0（无论 SPR 多少，没牌就是没牌）。中等成牌的“被压榨率”与 SPR 呈强负相关——SPR 越深，中等成牌在转牌/河牌被极化范围逼迫弃牌的概率越高，乘数越小。

```python
def correct_board_1_trips(dlev, position, bucket, draw, hand, spr):
    # 纯空气和弱听牌，几乎无法实现胜率
    if bucket in ["桶6:弱听牌", "桶7:纯空气"] and draw == "None":
        return dlev * 0.01  # 强制压缩到 1%
    
    # 中等成牌（如 JJ在 8-8-8 面），SPR越深，长线越被动
    if bucket == "桶3:中等成牌":
        # 假设 SPR=5 时，衰减到约 82% (对应之前 56 -> 46)
        # SPR 越大，扣减比例越多
        penalty_factor = 0.035 * spr
        multiplier = max(0.4, 1.0 - penalty_factor) # 设个底线防止扣成负数
        return dlev * multiplier
        
    return dlev

```

### 2. 对子彩虹面 (Paired Rainbow)

* **核心逻辑：** 防守方（OOP）中等成牌的反向隐含赔率随 SPR 增加而增大（乘数 $< 1$）。攻击方（IP）空气牌的多街诈唬权益同样随 SPR 增加而变大（因为深筹码才有空间开 3 枪，乘数 $> 1$）。

```python
def correct_board_2_paired_rainbow(dlev, position, bucket, draw, hand, spr):
    if position == "OOP" and bucket == "桶3:中等成牌":
        # 防守方中对，面临深筹连打压力
        penalty = 0.015 * spr  # SPR每增加1，额外缩水 1.5%
        return dlev * (1.0 - penalty)
        
    if position == "IP" and bucket == "桶7:纯空气":
        # 攻击方纯空气，深筹码赋予多街诈唬杠杆
        bonus = 0.30 * spr # 假设空气的原始DLEv很低，需要较高比例的乘数放大
        return dlev * (1.0 + bonus)
        
    return dlev

```

### 3. 对子两同花面 (Paired Two-Tone)

* **核心逻辑：** 湿润对子面对中等成牌的压迫力比彩虹面更强，同时必须引入花色阻挡判定。

```python
def correct_board_3_paired_twotone(dlev, position, bucket, draw, hand, spr, board_suits):
    if position == "OOP" and bucket == "桶3:中等成牌":
        # 湿润对子面，防守压力更大，衰减系数更高
        penalty = 0.02 * spr 
        return dlev * (1.0 - penalty)
        
    if bucket in ["桶6:弱听牌", "桶7:纯空气"]:
        # 检查是否阻挡了公牌的同花色（如公牌有两张草花，你手里有没有草花）
        has_blocker = any(suit in hand for suit in board_suits)
        if not has_blocker:
            # 没阻挡牌的弱牌，在深筹码下极易撞墙
            return dlev * max(0.5, 1.0 - (0.04 * spr))
            
    return dlev

```

### 4. 纯彩虹面 (Rainbow)

* **核心逻辑：** 这是典型的“隐含赔率驱动”牌面。大顺子听牌（OESD）在长线击中后，能够赢取的筹码量完全取决于 SPR。

```python
def correct_board_4_rainbow(dlev, position, bucket, draw, hand, spr):
    if draw == "OESD":
        # SPR 越大，隐含赔率越高（之前的绝对值+14 相当于 SPR=5 时放大了约 1.5倍）
        # 设定基数：SPR每深 1 个单位，听牌溢价增加 10%
        implied_odds_multiplier = 1.0 + (0.10 * spr)
        return dlev * implied_odds_multiplier
        
    return dlev

```

### 5. 两同花面 A (Two-Tone Type A - 高张两同花)

* **核心逻辑：** 顶对面临“被反超”的反向隐含赔率；而强同花听牌则享受隐含赔率。二者完全围绕筹码深度（SPR）反向拉扯。

```python
def correct_board_5_twotone_a(dlev, position, bucket, draw, hand, spr):
    if bucket == "桶2:中等顶对":
        # 筹码越深，中等顶对越不敢打光
        return dlev * max(0.6, 1.0 - (0.035 * spr))
        
    if draw == "FlushDraw":
        # 同花听牌买坚果，SPR越深越值钱
        return dlev * (1.0 + (0.05 * spr))
        
    return dlev

```

### 6. 两同花面 B (Two-Tone Type B - 低张两同花)

* **核心逻辑：** 低张两同花不仅有买花的威胁，还有出高张（A/K）的威胁。顶对的贬值速度比高张面还要快。

```python
def correct_board_6_twotone_b(dlev, position, bucket, draw, hand, spr):
    if bucket == "桶2:中等顶对":
        # 低张面上的顶对（如Q），极度害怕深筹码的转牌/河牌发高牌
        return dlev * max(0.5, 1.0 - (0.04 * spr))
        
    if draw == "FlushDraw":
        # 低张同花听牌，因为公牌点数低，经常还能兼顾买高张的活路
        return dlev * (1.0 + (0.06 * spr))
        
    return dlev

```

### 7. 单调面 A (Monotone Type A - 高张纯色)

* **核心逻辑：** 这种面的价值高度集中在“坚果同花阻挡牌（如手里单持同花 A 或 K）”。深筹码时，这张阻挡牌的诈唬（Bluff）溢价极高。

```python
def correct_board_7_monotone_a(dlev, position, bucket, draw, hand, spr, nut_suit):
    if bucket == "桶7:纯空气":
        # 判断手里是否持有最高/次高的单张同花色（假装自己有花）
        has_nut_blocker = (nut_suit in hand) and ("A" in hand or "K" in hand)
        if has_nut_blocker:
            # 深筹赋予了诈唬无敌的杠杆空间
            return dlev * (1.0 + (0.08 * spr))
        else:
            # 没花又没阻挡，筹码越深死得越快
            return dlev * max(0.2, 1.0 - (0.10 * spr))
            
    if draw == "FDPlusSD":
        # 狂暴听牌
        return dlev * (1.0 + (0.05 * spr))
        
    return dlev

```

### 8. 单调面 B (Monotone Type B - 低张纯色)

* **核心逻辑：** 对低张面的超强成牌（非同花暗三条），它的隐含赔率（买葫芦）和反向隐含赔率（对手已经成同花）极其撕裂。坚果听牌（As）则是绝对的王者。

```python
def correct_board_8_monotone_b(dlev, position, bucket, draw, hand, spr):
    # 坚果同花听牌
    if draw == "FlushDraw" and "As" in hand:
        # 绝对的坚果买牌，深筹变现能力呈指数级
        return dlev * (1.0 + (0.04 * spr))
        
    # 非同花的暗三条（买葫芦）
    if bucket == "桶1:超强成牌" and draw == "None":
        # 筹码足够深时，买葫芦能清空对手的小同花，SPR 极大地放大了这个收益
        # 若 SPR 很浅，反而没有这个额外收益
        return dlev * (1.0 + (0.035 * spr))
        
    return dlev

```

### 如何调整参数系数？

上面的代码中，诸如 `0.035`、`0.10` 等乘数斜率，是基于你原先 CSV 数据的差异反推的标准化参考值。

在你的解算器代码中，只需提取当前节点的有效筹码 `Stack`，当前 `Pot`，算出 `SPR = Stack / Pot`，传入函数即可。SPR 会自动根据底池的膨胀动态放大或缩小修正幅度。
