# 德州扑克 Postflop Solver — 开发交接文档

## 一、项目概述

本项目是基于 [postflop-solver](https://github.com/b-inary/postflop-solver) 开源库的二次开发，目标是将 Depth-Limited (DL) CFR 算法改造为实时 GTO 辅助解算器。

### 核心思想

```
完整 GTO 解算 (翻牌→转牌→河牌): 3-7 分钟 → 太慢
深度限制解算 (翻牌截断 + α修正): ~1 秒 → 实时可用
转牌圈/河牌圈: 全树展开 → 1-8 秒 (树很小)
```

---

## 二、架构

### 三条街的求解策略

| 街道 | 策略 | 耗时 | 精度 |
|------|------|:---:|:---:|
| **翻牌圈** | DL截断 + α post-solve 修正 | ~1s | MAE 10-14% |
| **转牌圈** | 全树展开（46张河牌） | 1-8s | 精确 |
| **河牌圈** | 全树展开（0张未知牌） | <0.1s | 精确 |

### 跨街范围传递

```
翻牌 DL → normalized_weights → 转牌全树展开 → normalized_weights → 河牌全树展开
```

通过 `PostFlopGame::set_custom_initial_weights()` 实现。

### 翻牌圈 DL 架构

```
第一阶段 (一次性预计算):
  ┌──────────────────────────────────────────────┐
  │ compute_equity_matrix()                      │
  │   → 预计算每对 (OOP手牌, IP手牌) 的all-in胜率│
  └──────────────────────────────────────────────┘

第二阶段 (CFR求解):
  ┌──────────────────────────────────────────────┐
  │ CFR在翻牌动作树上迭代                         │
  │   ├ 到达DL终端节点 → evaluate_depth_limited  │
  │   │   └ DLEv = Σ_v cfreach[v] × equity × factor│
  │   └ 2000次迭代后收敛                          │
  └──────────────────────────────────────────────┘

第三阶段 (求解后修正):
  ┌──────────────────────────────────────────────┐
  │ expected_values_with_board_correction()      │
  │   ├ detect_board_texture(flop) → 8种同构     │
  │   ├ classify_hand → 7种手牌桶                │
  │   └ DLEv × α(牌面,位置,桶)    ← α查找表     │
  └──────────────────────────────────────────────┘
```

## 四、α 系数体系

### 修正公式

```
修正后 DLEv = DLEv × α(牌面纹理, 位置, 手牌桶)

牌面纹理: 8种同构翻牌
位置: OOP / IP
手牌桶: 7种 (桶1:超强成牌 ~ 桶7:纯空气)
```

### 8种同构牌面

1. 三条面 (Trips): 3张同rank, 如 888
2. 对子彩虹 (Paired Rainbow): 2同rank + 3色
3. 对子两同花 (Paired Two-Tone): 2同rank + 2同花
4. 纯彩虹 (Rainbow): 3张不同rank + 3色
5. 两同花A (Two-Tone A): 2同花色包含最大rank
6. 两同花B (Two-Tone B): 2同花色不包含最大rank
7. 单调面A (Monotone A): 3同花色, 最大rank≥J
8. 单调面B (Monotone B): 3同花色, 最大rank<J

自动检测: `detect_board_texture(flop)`

### 7种手牌桶

1. 桶1:超强成牌 (Monster): Set/两对+/超强超对(AA/KK)
2. 桶2:中等顶对 (GoodTopPair): 顶对好踢脚
3. 桶3:中等成牌 (MediumMade): 中对/底对/弱超对
4. 桶4:超强强听牌 (MonsterDraw): FD+SD双抽/带对+前门花
5. 桶5:常规强听牌 (StrongDraw): 纯FD/纯OESD
6. 桶6:弱听牌 (WeakDraw): Gutshot/后门花
7. 桶7:纯空气 (Trash): 无对子无听牌

自动分类: `HandBucket::classify(cat, draw, hero, flop)`

### α 系数来源

α 系数从 8牌面×2SPR×2位置 的 Full EV 和未修正 DLEv 数据回归得到。对每组合 `α = mean(FullEV) / mean(DLEv)`，确保在所有 SPR 下有效。

## 六、已验证的效果

### EV 精度

| 模式 | MAE |
|------|:---:|
| 翻牌 Flop DL (无修正) | ~26% |
| 翻牌 Flop DL + α | **~10-14%** |
| 翻牌 Full solve | 0% (精确) |

### 速度

| 模式 | 耗时 |
|------|:---:|
| 翻牌 Full solve (SPR=20) | 100-460s |
| **翻牌 DL + α** | **~1s** (×300加速) |
| 转牌全树展开 (SPR=20) | 1-8s |
| 河牌全树展开 | <0.1s |

### 商业求解器对比

与 TexasSolver 在同一配置下 (QsJh2h, SPR=4, 60%bet):
- OOP 策略一致性: **99.4%**
- Full solve EV 精度: exploitability 0.22% vs 商业 0.45%
- 速度: 31x 更快 (7.9s vs 246s)

## 七、Fold Equity Bonus (实验性功能)

为解决 DL 模式下 IP bluff 频率不足的问题，在 `evaluate_depth_limited` 中加入了按桶差异化的 Fold Equity bonus：

```
fe_bonus = (1 - |equity|) × bucket_factor × opponent_air × half_pot
```

仅对 IP 生效（player==1），按桶分配不同权重:
- 桶6(弱听牌): 0.22 (最高,有backdoor潜力)
- 桶7(纯空气): 0.15 (次选)
- 桶3(中等成牌): 0.06

IP 平均 bet 频率从 26% 提升到 49%（接近 Full solve 的 54%），但策略仍偏二元。

## 八、Turn DL 尝试 (失败)

尝试了转牌截断方案 (depth_limit=Turn):
- 树膨胀到 500 节点 (Flop DL 的 50 倍)
- 耗时远超 Full solve 且无法收敛
- 结论: 从翻牌圈做 Turn 截断不可行，应回到 Flop DL 路径

## 九、使用方式

### 单牌面测试

```bash
cargo run --release --example bench_single -- \
  --flop "QsJh4d" \
  --pot 5 --stack 100 \
  --oop-range "AA,KK,..." --ip-range "QQ,JJ,..." \
  --bet-sizes "60%, e, a" \
  --output "result.csv"
```

### 8牌面批量测试

```bash
cargo run --release --example bench_multi_boards
```

### 修改 α 系数

编辑 `src/hand.rs` 中 `BoardCorrectionContext::flop_alpha()` 的查找表。

### IP 攻击因子

编辑 `src/hand.rs` 中 `BoardCorrectionContext::ip_aggression_factor()`。

### 修改 FE bonus

编辑 `src/game/evaluation.rs` 中 `evaluate_depth_limited` 的 `bucket_factor` 匹配。
